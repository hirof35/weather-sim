# Weather Simulator Go (Cgo-Free GUI)

Go言語の標準パッケージのみで構築された、環境依存のない（Cgoフリーな）リアルタイム気象情報シミュレーターです。
複雑なGUIライブラリやC言語への依存（Cgo）を一切排除し、Goの強力な並行処理（Goroutine）と標準のWebサーバー機能を組み合わせることで、どんな環境でも軽量かつ即座にブラウザ上で可視化画面を起動できます。

## 🚀 特徴

- **完全にCgoフリー:** 外部のC言語ライブラリに依存しないため、クロスコンパイルが非常に容易で、環境構築のトラブルがありません。
- **標準パッケージのみで完結:** `net/http` や `html/template` など、Goの標準ライブラリのみでリッチなWeb UI可視化を実現しています。
- **リアルタイムシミュレーション:** バックグラウンド処理（Goroutine）により、各都市の気温と天気が3秒ごとに自動で計算・更新されます。
- **スレッドセーフ設計:** `sync.Mutex` を採用し、データの更新処理とブラウザへの描画処理の間でデータ競合（Data Race）が起きないよう安全に管理しています。

## 🛠️ 動作環境

- Go 1.16 以上（標準パッケージのみ使用しているため、古いバージョンでも動作可能です）

## 📦 インストールと実行方法

外部ライブラリのインストール（`go get`）は一切不要です。

1. リポジトリをクローンするか、`main.go` を作成します。
2. ターミナルで以下のコマンドを実行します。

```bash
go run main.go
起動後、ブラウザで以下のURLを開いてください。
👉 http://localhost:8080

📊 画面イメージと仕組み
ブラウザを開くと、HTMLとCSSでデザインされたモダンなカード型のUIが表示されます。
HTMLの <meta http-equiv="refresh" content="3"> タグを利用し、ブラウザ側が3秒ごとに自動で最新の気象データをサーバーにリクエストして画面を更新します。

内部アーキテクチャ
メインスレッド: Webサーバー（http.ListenAndServe）を起動し、ユーザーからのアクセスを待ち受けます。

バックグラウンド（Goroutine）: メイン処理とは非同期で3秒ごとに都市データ（札幌・東京・沖縄）の気温・天候をランダムにシミュレート。

排他制御（Mutex）: データの読み書きが衝突しないよう、メモリ上のデータを安全に保護します。
